## 1. 项目定位与边界

### 1.1 核心定位

**韬纪元AI** 是一个 AI 助贷报告生产与资料作业平台，服务对象为助贷顾问、机构管理者和客户。

核心链路：
```
客户资料收集
  → AI 资料识别与缺口分析
  → 客户 360 画像
  → 顾问选择银行产品
  → AI 填写银行制式表格
  → 顾问逐字段复核
  → 导出材料包（ZIP）
  → 提交银行并记录调用
```

### 1.2 合规边界（开发必读）

| 约束 | 说明 |
|------|------|
| 不放款 | 平台不提供资金、不承诺额度 |
| 不审批 | 不替代银行及持牌金融机构的最终审核 |
| 必复核 | 所有 AI 输出结果均需顾问人工复核后才能用于提交 |
| 不盲选 | C 端客户不能自行选择银行产品，须由顾问操作 |

## 2. 系统整体架构

### 2.1 四端入口

```
┌─────────────────────────────────────────────────────┐
│                   韬纪元AI 平台                        │
├──────────────┬──────────────┬──────────────┬────────┤
│  B端PC工作台  │  C端客户入口  │  顾问移动端   │ 管理后台│
│ (Web/React)  │(微信小程序/H5)│(微信小程序/H5)│(Web)  │
└──────────────┴──────────────┴──────────────┴────────┘
                        │
              ┌──────────────────┐
              │   后端 REST API   │
              │  (Node.js/Java)  │
              └──────────────────┘
                        │
         ┌──────────────┼──────────────┐
         │              │              │
    ┌────────┐   ┌─────────────┐  ┌────────┐
    │ MySQL  │   │  AI 服务层   │  │ 文件存储│
    │(主数据)│   │(识别/填表/AI)│  │(OSS/COS│
    └────────┘   └─────────────┘  └────────┘
```

### 2.2 技术栈建议

| 层 | 技术选型 | 说明 |
|----|---------|------|
| B端PC前端 | React + TypeScript + Ant Design | SPA，路由用 react-router |
| C端/顾问移动端 | 微信小程序（已有 miniprogram_c 框架，Skyline 渲染） | TypeScript + glass-easel |
| 后端 | Node.js (NestJS) 或 Java (Spring Boot) | RESTful API + JWT 认证 |
| 数据库 | MySQL 8.0 | 主业务数据 |
| 文件存储 | 腾讯云 COS | 客户资料、导出材料包 |
| AI 能力 | 第三方 LLM API（GPT-4/Claude/国内大模型） + OCR | 资料识别、表格填写 |
| 缓存 | Redis | 会话、任务队列状态 |
| 消息队列 | Redis Queue 或 RabbitMQ | AI 异步任务 |

## 5. MySQL 数据库表设计

> 字符集：utf8mb4；时间字段统一使用 `DATETIME`；软删除用 `deleted_at`

### 5.1 账号与权限

```sql
-- 机构表
CREATE TABLE institutions (
  id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  name          VARCHAR(100)    NOT NULL COMMENT '机构名称',
  plan_id       INT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '当前套餐ID',
  quota_total   INT             NOT NULL DEFAULT 30 COMMENT '当月总配额',
  quota_used    INT             NOT NULL DEFAULT 0  COMMENT '当月已用次数',
  quota_reset_at DATE           COMMENT '配额重置日期',
  status        TINYINT         NOT NULL DEFAULT 1 COMMENT '1=启用 0=停用',
  created_at    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at    DATETIME        DEFAULT NULL
) ENGINE=InnoDB COMMENT='机构表';

-- 用户表（顾问/主管/运营管理员）
CREATE TABLE users (
  id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  institution_id BIGINT UNSIGNED NOT NULL COMMENT '所属机构',
  name          VARCHAR(50)     NOT NULL COMMENT '姓名',
  phone         VARCHAR(20)     UNIQUE NOT NULL COMMENT '手机号（登录账号）',
  password_hash VARCHAR(255)    NOT NULL COMMENT 'bcrypt',
  wx_openid     VARCHAR(64)     DEFAULT NULL COMMENT '微信小程序openid',
  wx_unionid    VARCHAR(64)     DEFAULT NULL,
  role          ENUM('advisor','supervisor','admin') NOT NULL DEFAULT 'advisor',
  data_scope    ENUM('self','team','all')            NOT NULL DEFAULT 'self'
                COMMENT '仅本人/本团队/全部客户',
  status        TINYINT         NOT NULL DEFAULT 1 COMMENT '1=启用 0=停用',
  last_login_at DATETIME        DEFAULT NULL,
  created_at    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at    DATETIME        DEFAULT NULL,
  INDEX idx_institution (institution_id),
  INDEX idx_phone (phone)
) ENGINE=InnoDB COMMENT='系统用户表';

-- 用户功能权限表（精细权限）
CREATE TABLE user_permissions (
  id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id     BIGINT UNSIGNED NOT NULL,
  permission  VARCHAR(50)     NOT NULL
              COMMENT '如: view_customer, edit_followup, ai_parse, maintain_bank_material, manage_plan, manage_account, config_permission',
  INDEX idx_user (user_id)
) ENGINE=InnoDB COMMENT='用户功能权限表';
```

### 5.3 资料管理

```sql

  created_at    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_document (document_id),
  INDEX idx_customer (customer_id)

  institution_id BIGINT UNSIGNED NOT NULL,
  customer_id   BIGINT UNSIGNED NOT NULL,
  trigger_user_id BIGINT UNSIGNED NOT NULL,
  scope         ENUM('single_doc','all_docs','reparse') DEFAULT 'all_docs',
  document_ids  JSON            DEFAULT NULL COMMENT '指定文件IDs，NULL=全部',
  status        ENUM('queued','processing','done','failed') DEFAULT 'queued',
  result_summary TEXT           DEFAULT NULL COMMENT '缺口摘要',
  started_at    DATETIME        DEFAULT NULL,
  finished_at   DATETIME        DEFAULT NULL,
  created_at    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
```

### 5.5 报告作业台（AI 填表）

```sql
-- AI 填表任务表
CREATE TABLE report_tasks (
  id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  institution_id  BIGINT UNSIGNED NOT NULL,
  customer_id     BIGINT UNSIGNED NOT NULL,
  advisor_id      BIGINT UNSIGNED NOT NULL,
  product_id      INT UNSIGNED    NOT NULL COMMENT '目标银行产品',
  status          ENUM('pending','ai_filling','ai_done','reviewing','review_done','exporting','exported','submitted')
                  NOT NULL DEFAULT 'pending',
  ai_fill_at      DATETIME        DEFAULT NULL,
  reviewed_at     DATETIME        DEFAULT NULL,
  exported_at     DATETIME        DEFAULT NULL,
  submitted_at    DATETIME        DEFAULT NULL COMMENT '提交银行时间',
  export_url      VARCHAR(500)    DEFAULT NULL COMMENT '导出材料包URL',
  issue_count     INT             DEFAULT 0 COMMENT 'AI标记的问题字段数',
  created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_customer (customer_id),
  INDEX idx_advisor (advisor_id),
```

## 7. AI 能力层说明

### 7.1 AI 任务类型

| 类型 | 触发场景 | 消耗配额 | 输入 | 输出 |
|------|---------|---------|------|------|
| 资料识别 | 客户上传资料后 / 顾问手动触发 | 1次/份 | 文件（图片/PDF） | 字段kv + 缺口分析 |
| 客户画像 | 首次识别完成后自动生成 | 不额外扣 | 所有识别结果 | 摘要+风险点+标签 |
| AI填表 | 顾问在报告作业台创建任务 | 1次/任务 | 客户字段+银行产品配置 | 字段草稿+问题标注 |
| 补件清单 | 顾问在AI助手面板请求 | 不扣次 | 客户资料缺口 | 自然语言补件指引 |
| 问题字段重填 | 顾问标注问题后 | 不额外扣 | 顾问标注+原草稿 | 更新后的字段值 |

### 7.2 异步任务处理模式

```
POST /ai/recognize
  → 写入 ai_recognition_tasks（status=queued）
  → 加入 Redis 队列
  → 返回 task_id

Worker 消费队列:
  → 调用 OCR/LLM API
  → 写入 ai_recognition_results
  → 更新 customer_documents.ai_parse_status = done
  → 更新 customers.doc_completeness
  → 推送进度通知

前端轮询 GET /ai/recognize/:taskId 或 WebSocket 推送
```
