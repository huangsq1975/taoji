## 1. 项目定位与边界

### 1.1 核心定位

**韬纪元AI** 是一个 AI 助贷报告生产与资料作业平台，服务对象为助贷顾问、机构管理者和客户。

核心链路：
```
客户资料收集
  → AI 资料识别与缺口分析
  → 客户 360 画像
  → 顾问选择银行产品
  → AI 填写银行制式表格
  → 顾问逐字段复核
  → 导出材料包（ZIP）
  → 提交银行并记录调用
```

### 1.2 合规边界（开发必读）

| 约束 | 说明 |
|------|------|
| 不放款 | 平台不提供资金、不承诺额度 |
| 不审批 | 不替代银行及持牌金融机构的最终审核 |
| 必复核 | 所有 AI 输出结果均需顾问人工复核后才能用于提交 |
| 不盲选 | C 端客户不能自行选择银行产品，须由顾问操作 |

```
    ┌──────────┐ ┌─────────────┐  ┌────────┐
```

### 2.2 技术栈

| 层 | 技术选型 | 说明 |
|----|---------|------|
| B端PC前端 | React 18.3 + TypeScript 5.5 + Vite 5.4 + React Router 6.26 | SPA，纯 CSS 自定义设计系统（无第三方 UI 库），strict 模式 |

| 异步任务 | Spring `@Async` + 线程池 | AI 识别任务异步处理 |
| 包管理 | Maven | `backend-java/pom.xml` |

```
Base URL: /api/v1
认证: Authorization: Bearer <JWT_TOKEN>
响应格式:
{
  "statusCode": 200,
  "message": "ok",
  "data": {}
}
分页参数: ?page=1&pageSize=20
```

## 7. AI 能力层说明

### 7.1 AI 任务类型

| 类型 | 触发场景 | 消耗配额 | 当前状态 |
|------|---------|---------|---------|
| 资料识别 | 文件上传后 / 顾问手动触发 | 1次/份 | 🔴 **Mock**（`getMockRecognitionResults()`） |
| 客户画像 | 识别完成后自动生成 | 不额外扣 | 🔴 **Mock** |
| AI填表 | 顾问在报告作业台创建任务 | 1次/任务 | 🔴 **Mock 占位**（`ReportService.createReportTask()`） |
| AI对话 | C端客户/顾问移动端 | 不扣次 | 🟡 **有真实 API 框架，无 Key 时 Mock 兜底** |
| 补件清单 | AI助手面板（未实现） | 不扣次 | ❌ 功能入口未实现 |

### 7.2 配置项

```yaml
# application.yml
ai:
  api-url: https://api.openai.com/v1   # 可通过 AI_API_URL 环境变量覆盖
  api-key: ""                          # 通过 AI_API_KEY 环境变量注入
  model: gpt-4o                        # 通过 AI_MODEL 环境变量覆盖
```

### 7.3 异步任务处理流程

```
POST /ai/recognize
  → 写入 ai_recognition_tasks（status=queued）
  → Spring @Async 线程池异步执行
  → 返回 task_id

异步 Worker（processRecognitionAsync）:
  → 更新 status=processing
  → 调用 OCR/LLM API（当前为 Mock）
  → 写入 ai_recognition_results
  → 更新 customer_documents.ai_parse_status=done
  → 更新 customers.doc_completeness

前端轮询 GET /ai/recognize/:taskId 查询状态
```
